## Summary

Four hardening changes required before the first public release: (A) pin the ALAC FetchContent dependency to a reproducible commit SHA, (B) add a PR/push CI workflow (`.github/workflows/ci.yml`) that builds the debug preset and runs unit tests, (C) fix the appcast `publish_dir` TODO in `release.yml` by removing the redundant Step 10, and (D) fix the code-signing `if:` guard so it evaluates correctly at job level. A fifth sub-task emerges from research: `CMakePresets.json` hardcodes absolute local MSVC paths that do not exist on GitHub-hosted runners — CI-compatible presets must be added.

## Phase 0: Research Findings

### Decision 1 — ALAC Pinned SHA

- **Decision**: Pin to `c38887c5c5e64a4b31108733bd79ca9b2496d987`
- **Rationale**: This is HEAD of `macosforge/alac` master as of 2026-03-22. The repo has had no commits since 2016-05-11 and has zero tags or releases. `master` is effectively frozen; pinning to the SHA is correct hygiene and documents exactly what we build against.
- **Alternatives considered**: No tags exist on this repo; no tag-based alternative is available.

### Decision 2 — CMakePresets.json Portability

- **Decision**: Add a hidden `base-ci` preset that inherits `base` and overrides all absolute compiler/linker paths with bare binary names (`cl.exe`, `link.exe`, `rc.exe`, `mt.exe`), and clears the hardcoded `PATH`/`LIB`/`INCLUDE` env blocks. Add `msvc-x64-debug-ci` configure/build/test presets inheriting `base-ci`. Use `ilammy/msvc-dev-cmd@v1` in `ci.yml` to populate the environment before CMake runs.
- **Rationale**: Keeps the existing `base` preset and all local developer builds unchanged. CI uses a dedicated preset with environment provided by the workflow.
- **Alternatives considered**: Strip paths from `base` directly (breaks developer builds); override compilers on the cmake command line (fragile, undocumented); rely on env vars alone without a dedicated preset (cached CMakeCache.txt may retain the absolute path after first configure).

### Decision 3 — Codesign Guard Fix

- **Decision**: Move `CODESIGN_PFX` and `CODESIGN_PASSWORD` secrets to the **job-level `env:`** block. The existing `if: env.CODESIGN_PFX != ''` condition then evaluates correctly from the job environment.
- **Rationale**: In GitHub Actions, step-level `env:` is populated after the `if:` expression is evaluated. Current code sets `CODESIGN_PFX` only in the step env so the guard is always false — signing **never runs** even when the secret is set. Job-level env fixes this with minimal diff.
- **Alternatives considered**: `if: ${{ secrets.CODESIGN_PFX != '' }}` — unreliable in some runner versions; job-level env is the documented safe pattern.

### Decision 4 — Appcast `publish_dir` Fix

- **Decision**: **Remove Step 10** (`peaceiris/actions-gh-pages@v4` with `publish_dir: ./docs`) entirely. Step 9 already pushes `appcast.xml` directly to `gh-pages` via `git push`. No `docs/` directory exists in the repository.
- **Rationale**: Step 10 is dead scaffolding. Running it would overwrite `gh-pages` content from an empty directory, corrupting the appcast committed by Step 9.
- **Alternatives considered**: Change `publish_dir` to some real path — no website content directory exists to publish.
