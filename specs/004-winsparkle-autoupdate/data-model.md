# Data Model: WinSparkle Auto-Update Key Pair

**Feature**: 004-winsparkle-autoupdate  
**Date**: 2026-03-22

---

## Key Entities & Files

### `resources/sparkle_pubkey.txt`
Documentation file (not parsed by build). Contains human-readable instructions and the canonical Base64 Ed25519 public key string for reference. The key is also embedded in each RC file.

| Field | Value |
|-------|-------|
| Format | UTF-8 plain text |
| Committed to repo | ✅ Yes |
| Contains secrets | ❌ No |
| Used at build time | ❌ No (documentation only) |
| Updated when | Key is rotated |

---

### `resources/locales/strings_*.rc` (7 files)

Each locale RC file contains `IDS_SPARKLE_PUBKEY` with the Base64 Ed25519 public key as a string literal. This is the authoritative compile-time source.

| Field | Value |
|-------|-------|
| Files | `strings_en.rc`, `strings_de.rc`, `strings_fr.rc`, `strings_es.rc`, `strings_ja.rc`, `strings_ko.rc`, `strings_zh-Hans.rc` |
| Key format | Base64-encoded Ed25519 public key (≥ 43 characters) |
| Current value | `TODO_REPLACE_WITH_ED25519_PUBLIC_KEY` (placeholder) |
| Embedded into | `AirBeam.exe` via RC compilation |
| Updated when | Key is rotated |

### `SPARKLE_PRIVATE_KEY` (GitHub Actions secret)

The Ed25519 private key used to sign release installers. Never stored anywhere except the GitHub repository secrets store.

| Field | Value |
|-------|-------|
| Format | Base64-encoded Ed25519 private key (64 chars) |
| Storage | GitHub repository secrets (`Settings → Secrets → Actions`) |
| Used in | `release.yml` Step 7 (signing step) |
| Passed as | `env.SPARKLE_PRIVATE_KEY` at job level |
| Generated by | `winsparkle-sign.exe --generate-keys` |
| Committed to repo | ❌ Never |

---

### `winsparkle-sign.exe`

CLI tool bundled in the WinSparkle GitHub release zip.

| Field | Value |
|-------|-------|
| Source | `https://github.com/vslavik/winsparkle/releases/download/v{VERSION}/WinSparkle-{VERSION}.zip` |
| Pinned version | `v0.8.5` (controlled by `WINSPARKLE_VERSION` env var in release.yml) |
| Key generation | `winsparkle-sign.exe --generate-keys` |
| Signing | `winsparkle-sign.exe <installer.exe>` (reads `SPARKLE_PRIVATE_KEY` from env) |
| Verification | `winsparkle-sign.exe --verify <installer.exe>` |
| Output format | Base64 string on stdout |

### `.github/appcast.xml` (seed template)

Committed to repo. Used to seed the `gh-pages` branch's `appcast.xml` on first deploy. Updated by the release workflow on every release.

**Schema** (with correct `sparkle:edSignature` placement on `<enclosure>`):

```xml
<?xml version="1.0" encoding="utf-8"?>
<rss version="2.0" xmlns:sparkle="http://www.andymatuschak.org/xml-namespaces/sparkle">
  <channel>
    <title>AirBeam Changelog</title>
    <link>https://drsvenkuehn.github.io/airbeam/appcast.xml</link>
    <description>Most recent changes to AirBeam</description>
    <language>en</language>
    <item>
      <title>AirBeam {VERSION}</title>
      <pubDate>{RFC-822 date}</pubDate>
      <sparkle:version>{VERSION}</sparkle:version>
      <sparkle:shortVersionString>{VERSION}</sparkle:shortVersionString>
      <enclosure
        url="https://github.com/drsvenkuehn/airbeam/releases/download/v{VERSION}/AirBeam-v{VERSION}-Setup.exe"
        sparkle:edSignature="{BASE64_EDDSA_SIGNATURE}"
        sparkle:version="{VERSION}"
        type="application/octet-stream"
        length="{BYTES}"
      />
      <sparkle:releaseNotesLink>
        https://github.com/drsvenkuehn/airbeam/releases/tag/v{VERSION}
      </sparkle:releaseNotesLink>
    </item>
  </channel>
</rss>
```

**Key constraint**: `sparkle:edSignature` MUST be an attribute of `<enclosure>`, not a child element of `<item>`. WinSparkle validates the signature of the download URL referenced by `<enclosure>`.

### `appcast.xml` (on `gh-pages` branch)

Served by GitHub Pages at `https://drsvenkuehn.github.io/airbeam/appcast.xml`. This is what WinSparkle polls.

| Field | Value |
|-------|-------|
| Location | Root of `gh-pages` branch |
| Updated by | `release.yml` Step 9 via `peaceiris/actions-gh-pages@v4` |
| Accumulates | All release entries (newest first is conventional but WinSparkle picks the highest `sparkle:version`) |
| GitHub Pages URL | `https://drsvenkuehn.github.io/airbeam/appcast.xml` |

## State Transitions

```
Key Pair State:
  [Not generated] 
      → (developer runs winsparkle-sign --generate-keys)
  [Keys generated, local]
      → (pubkey committed to RC files; privkey added to GitHub secret)
  [Active - live in binary]
      → (key compromised or rotated)
  [Rotated - new key committed, old key abandoned]
```

```
appcast.xml State:
  [Seed template in .github/appcast.xml]  ← committed to main
      → (first v* tag pushed → release workflow runs)
  [Live on gh-pages with v1.0.0 entry]
      → (next v* tag pushed)
  [Live on gh-pages with v1.0.1 entry prepended]
      → ...
```
