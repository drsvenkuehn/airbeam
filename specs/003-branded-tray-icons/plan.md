# Implementation Plan: Branded Tray Icons

**Branch**: `main` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)

---

## Summary

Replace 11 single-frame placeholder `.ico` files (~680 bytes each) with real branded artwork:
- 3 static states (idle, streaming, error) + 8 connecting-animation frames
- Each ICO contains 16×16, 32×32, 48×48 frames in 32-bit RGBA
- SVG source files committed to `resources/icons/src/`
- One code fix: `ANIM_INTERVAL_MS` 150 → 125 ms (SC-004 bug in `TrayIcon.cpp`)
- One tooling addition: PowerShell validation script + CTest test

**No protocol or audio code changes**. All existing RC/resource-ID infrastructure is already correct.

---

## Implementation Phases

### Phase 1 — Tooling Scripts (No design required)

Create the PowerShell scripts that wrap ImageMagick and automate the export/validation workflow.
These can be written and committed before any SVG artwork exists.

1. `resources/icons/export_icons.ps1` — reads SVGs from `src/`, exports multi-frame ICOs
2. `resources/icons/validate_icons.ps1` — validates all 11 ICOs meet the contract
3. `resources/icons/src/gen_frames.ps1` — rotates `icon_connecting_base.svg` to produce 8 frames

---

### Phase 2 — SC-004 Code Fix

Fix the one-line animation interval bug in `TrayIcon.cpp`:
```cpp
// Before
static constexpr UINT  ANIM_INTERVAL_MS = 150;
// After
static constexpr UINT  ANIM_INTERVAL_MS = 125;
```

Build and confirm no regressions.

---

### Phase 3 — CTest Icon Validation Test

Add a CTest test that runs `validate_icons.ps1` and fails if any ICO is a placeholder.
Initially this test will **fail** (placeholders present) — it defines the acceptance criterion.
The test goes green when Phase 4 is complete.

---

### Phase 4 — SVG Design

Create the 4 base SVG files in `resources/icons/src/`:
1. `icon_idle.svg` — gray outlined speaker + arcs
2. `icon_streaming.svg` — green filled speaker + arcs
3. `icon_error.svg` — red speaker + X overlay
4. `icon_connecting_base.svg` — blue speaker + single arc at 0°

Run `gen_frames.ps1` to produce `icon_connecting_001.svg` – `_008.svg`.

**This phase requires human design work** — the agent cannot produce visual artwork.

---

### Phase 5 — ICO Export and Commit

Run `export_icons.ps1` to produce all 11 ICO files, verify with `validate_icons.ps1`,
do a manual visual review in the running tray app, then commit.

---

### Phase 6 — Final Validation

- Build Release configuration
- Run `ctest` (icon-validation test must pass)
- Manual SC-001–SC-004 review (visual check at 16×16 and 32×32, connecting animation timing)

## Decision Log

| Decision | Choice | Rationale |
|----------|--------|-----------|
| ICO generation tool | ImageMagick offline | No CI complexity; ICOs are pre-committed binary assets |
| SVG frame generation | Script-based rotation (gen_frames.ps1) | Ensures visual consistency across 8 connecting frames |
| Icon design concept | Speaker + wireless arcs, 4-state color+shape matrix | Legible at 16×16; accessible without color |
| ICO frames | 16×16 + 32×32 + 48×48, 32-bit RGBA | Required for DPI awareness and taskbar pinning |
| ANIM_INTERVAL_MS | 125 ms (was 150) | SC-004 compliance: 8 × 125 = 1000 ms cycle |
| CI validation | CTest + PowerShell size/frame check | Catches placeholder regressions without design tooling in CI |
| No proprietary design format | SVG only | Open format; reproducible by all contributors |
