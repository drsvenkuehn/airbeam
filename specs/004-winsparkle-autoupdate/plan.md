## Implementation Phases

### Phase 1 — One-Time Key Generation (Developer)
Performed once on developer machine before any CI changes. See `quickstart.md`.

1. Download WinSparkle 0.8.5 zip; extract `winsparkle-sign.exe`
2. Run `winsparkle-sign.exe --generate-keys` → capture public and private key
3. Add `SPARKLE_PRIVATE_KEY` to GitHub repository secrets
4. Replace placeholder in all 7 `resources/locales/strings_*.rc` files
5. Update `resources/sparkle_pubkey.txt` with real key + generation date

**Verification**: SC-005 — `sparkle_pubkey.txt` and all RC files contain no placeholder; key length ≥ 43 chars.

---

### Phase 2 — Fix `.github/appcast.xml` Template
Fix the existing template file that the release workflow reads to seed appcast.xml.

1. Move `sparkle:edSignature` from `<item>` child element → `<enclosure>` attribute
2. Replace all `TODO_ORG` → `drsvenkuehn`
3. Update installer filename to match release.yml naming convention: `AirBeam-v{VERSION}-Setup.exe`

---

### Phase 3 — Fix `release.yml`: Add WinSparkle Download Step
Add a new step before all others that downloads WinSparkle and makes both `WinSparkle.dll` and `winsparkle-sign.exe` available.

```yaml
- name: Download WinSparkle binaries
  shell: pwsh
  run: |
    $ver = $env:WINSPARKLE_VERSION
    $url = "https://github.com/vslavik/winsparkle/releases/download/v$ver/WinSparkle-$ver.zip"
    Invoke-WebRequest $url -OutFile WinSparkle.zip
    Expand-Archive WinSparkle.zip -DestinationPath WinSparkle-bin
    $sign = Get-ChildItem WinSparkle-bin -Recurse -Filter "winsparkle-sign.exe" | Select-Object -First 1
    $dll  = Get-ChildItem WinSparkle-bin -Recurse -Filter "WinSparkle.dll"      | Select-Object -First 1
    Copy-Item $sign.FullName "winsparkle-sign.exe"
    Copy-Item $dll.FullName  "WinSparkle.dll"
    echo "$PWD" >> $env:GITHUB_PATH
```

Add `WINSPARKLE_VERSION: "0.8.5"` to job-level `env:` block.


### Phase 5 — Fix `release.yml`: appcast.xml Generation (Step 9)
Replace the brittle manual git push with `peaceiris/actions-gh-pages@v4` and fix `sparkle:edSignature` placement.

Key changes:
- `sparkle:edSignature` moves to be an attribute of `<enclosure>` (not child of `<item>`)
- Replace manual `git checkout gh-pages / git push` with `peaceiris/actions-gh-pages@v4`
- On first run: `gh-pages` branch is created automatically by the action if it doesn't exist
- The appcast.xml is built from scratch each release (read seed from `.github/appcast.xml`, inject new item with correct signature)


## Decision Log

| Decision | Choice | Rationale |
|----------|--------|-----------|
| `winsparkle-sign` invocation | `winsparkle-sign.exe <binary>` reads `SPARKLE_PRIVATE_KEY` from env | Avoids writing key to disk |
| `sparkle:edSignature` placement | Attribute of `<enclosure>`, not child of `<item>` | WinSparkle spec requirement |
| WinSparkle download | Pinned `v0.8.5` zip from GitHub releases | Reproducibility + gets both DLL and sign tool |
| gh-pages deployment | `peaceiris/actions-gh-pages@v4` | Standard action; handles branch creation + force-push |
| Public key embedding | RC string resource `IDS_SPARKLE_PUBKEY` | Existing correct implementation; no code change needed |
| SPARKLE_PRIVATE_KEY guard | Job-level `env:` pattern (spec 002 lesson) | Step-level `env:` evaluated after `if:` — always false |
