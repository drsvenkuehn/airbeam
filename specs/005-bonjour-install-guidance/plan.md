# Implementation Plan: Bonjour Install Guidance

**Branch**: `005-bonjour-install-guidance` | **Date**: 2026-03-23 | **Spec**: [spec.md](spec.md)

---

## Summary

The Bonjour-missing notification infrastructure already exists: `BonjourLoader` detects a missing `dnssd.dll` and posts `WM_BONJOUR_MISSING`, which calls `BalloonNotify::ShowWarning`. However four gaps prevent the feature from meeting its spec:

1. Balloon body text lacks the Apple download URL (FR-001, FR-006)
2. `NIN_BALLOONUSERCLICK` is not handled — balloon click does nothing (FR-007)
3. Title and body use the same string resource ID — no distinct "AirBeam — Bonjour Required" title (FR-005)
4. Tray tooltip in error state says "AirBeam — Error" with no Bonjour mention (FR-008)

**No new threads, no new architecture.** All work is resource strings + ~20 lines of C++ on Thread 1 (UI message loop). No audio or protocol paths are touched.

## Implementation Phases

### Phase 1 — Resource IDs and String Updates

Add the two missing resource IDs and update all 7 locale RC files:

**`resource_ids.h`** — add after existing tooltip IDs:
```cpp
#define IDS_TOOLTIP_BONJOUR_MISSING   1005   // "AirBeam — Bonjour Not Found"
```
And after existing balloon IDs:
```cpp
#define IDS_BALLOON_TITLE_BONJOUR_MISSING  1016  // "AirBeam — Bonjour Required"
```

**`strings_en.rc`** updates:
```rc
IDS_TOOLTIP_BONJOUR_MISSING      "AirBeam \x2014 Bonjour Not Found"
IDS_BALLOON_TITLE_BONJOUR_MISSING "AirBeam \x2014 Bonjour Required"
IDS_BALLOON_BONJOUR_MISSING       "Bonjour is not installed. AirPlay speaker discovery requires Bonjour. Visit: https://support.apple.com/downloads/bonjour-for-windows"
```

**Translation strategy for 6 non-English locales** (constitution §VIII: no missing or empty values at release):
- The Apple URL (`https://support.apple.com/downloads/bonjour-for-windows`) is identical in all locales — only surrounding instructional text is translated.
- The 6 non-English locale files already contain a translated `IDS_BALLOON_BONJOUR_MISSING` body. These must be updated to append the URL.
- `IDS_BALLOON_TITLE_BONJOUR_MISSING` and `IDS_TOOLTIP_BONJOUR_MISSING` are new — each locale file must add both keys. Until community-reviewed translations are available, **use the English string verbatim as the fallback value** (annotated with a comment so translators can find and update them).
- Example fallback pattern (de, fr, es, ja, ko, zh-Hans):
  ```rc
  IDS_TOOLTIP_BONJOUR_MISSING      "AirBeam \x2014 Bonjour Not Found"     // TODO(i18n): translate
  IDS_BALLOON_TITLE_BONJOUR_MISSING "AirBeam \x2014 Bonjour Required"     // TODO(i18n): translate
  ```
- The balloon body strings already exist in translated form and only need the URL appended.

### Phase 2 — C++ Handler Fixes

**`AppController::HandleBonjourMissing()`** — fix title/body split and add Bonjour-specific state:
```cpp
void AppController::HandleBonjourMissing() {
    trayIcon_.SetState(TrayState::BonjourMissing);   // sets error icon + Bonjour tooltip atomically
    lastBalloonWasBonjour_ = true;
    balloonNotify_.ShowWarning(IDS_BALLOON_TITLE_BONJOUR_MISSING,
                               IDS_BALLOON_BONJOUR_MISSING);
}
```

**`AppController::HandleTrayCallback()`** — add `NIN_BALLOONUSERCLICK`:
```cpp
case NIN_BALLOONUSERCLICK:
    if (lastBalloonWasBonjour_) {
        ShellExecuteW(nullptr, L"open",
                      BONJOUR_DOWNLOAD_URL,
                      nullptr, nullptr, SW_SHOWNORMAL);
        lastBalloonWasBonjour_ = false;
    }
    break;
```

**`AppController.h`** — add flag:
```cpp
bool lastBalloonWasBonjour_ = false;
```

### Phase 3 — CTest SC-002 URL Validation

Add a PowerShell-based CTest that verifies the Apple URL returns HTTP 200:

```cmake
find_program(PWSH_EXECUTABLE NAMES pwsh pwsh.exe)
if(PWSH_EXECUTABLE)
    add_test(
        NAME    bonjour-url-check
        COMMAND "${PWSH_EXECUTABLE}" -Command
                "try { $r = Invoke-WebRequest 'https://support.apple.com/downloads/bonjour-for-windows' -Method Head -UseBasicParsing -TimeoutSec 15; if ($r.StatusCode -ge 200 -and $r.StatusCode -lt 400) { exit 0 } else { exit 1 } } catch { exit 1 }"
    )
    set_tests_properties(bonjour-url-check PROPERTIES LABELS "unit")
endif()
```

## Decision Log

| Decision | Choice | Rationale |
|----------|--------|-----------|
| URL in balloon body | Embed URL text directly in `IDS_BALLOON_BONJOUR_MISSING` | `NOTIFYICONDATA.szInfo` is plain text — no hyperlink support in Win32 balloons; URL must be readable text |
| Stable Apple URL | `https://support.apple.com/downloads/bonjour-for-windows` | Support article URL — stable across Bonjour version changes; HTTP 200 confirmed |
| `lastBalloonWasBonjour_` flag | `bool` on `AppController` | Only one balloon type needs URL routing; avoids over-engineering a generic click-dispatch system. **Known limitation**: if a second balloon fires after the Bonjour balloon (e.g. `HandleRaopFailed`) and the user then clicks it, the flag is still `true` and the Bonjour URL opens. Acceptable for v1.0 given the rarity of concurrent errors on first run. |
| Tooltip on Bonjour error | `IDS_TOOLTIP_BONJOUR_MISSING` distinct from `IDS_TOOLTIP_ERROR` | FR-008: keyboard-only users must be able to discover the issue via tooltip; generic "Error" gives no guidance |
| CTest label | `"unit"` | Runs automatically in CI via `ctest --preset msvc-x64-debug-ci`; GitHub Actions runners have internet access; HEAD request is fast (~1–2 s) and deterministic against a stable Apple support URL. |
